## Referencing CSS
Our CSS file needs to be referenced in our HTML. The best practice is to include all of our styles in a single external style sheet, which is then referenced from within the `<head>` element of our HTML document. Other options for referencing CSS include using internal and inline styles, but they are generally frowned upon.

Within the `<head>` element, the `<link>` element is used to define the relationship between the HTML file and the CSS file. Since we are linking to CSS, use the `rel` attribute with a value of `stylesheet` to specify their relationship, as well as the `href` attribute to identify the localtion of the CSS file.

```html
<head>
  <link rel="stylesheet" href="main.css">
</head>
```

Some elements have default styles without being defined by CSS. This is the browser imposing its own preferred CSS styles for those elements.

## Using CSS Resets
Every web browser has its own default styles for different elements. To ensure cross-browser compatibility, CSS resets are common. CSS resets take every common HTML element with a predefined style and provide one unified style for all browsers. Because CSS cascades from top to bottom, our reset needs to be at the very top of our style sheet.
